Buildbot for ICON

Buildbot starts every night a test suite on different machines, using different compilers.

The test consists in:

  1. removing files from the previous test
  2. retrieving the most recent revision existing on "trunk/icon-dev"
  3. running "configure&make" for a specifed target machine and compiler
  4. making test runs with the following sub steps:
    1. make run scripts for the target machine and for all experiments, for which descriptor files named run/exp.test_* exist
    2. launch the test run scripts
    3. launch existing post processing scripts, named run/post_*

The success (in green) or failure (in red) of these tests is displayed on the following pages:

  • Waterfall display: Displays success/failure for all 4 steps listed above for all build targets, in chronological order.
  • Grid display: Displays overall success or failure for the 20 most recent revisions tested by Buildbot, including regular nightly builds and additional test builds, ordered by build target.
  • Transposed Grid Display: Presents the same information as the grid, but lists the revisions down the side.
  • Latest build: Displays the overall success or failure for the last test of each builder, ordered by builder.
  • List display: Displays the overall success or failure of the most recent builds, ordered by time of completion of the build tests.
  • Build slaves: Displays the status of all machines used by this Buildbot.
